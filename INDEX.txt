🔌 API USAGE EXAMPLES
═══════════════════════════════════════════════════════════════════════

📤 Encrypt a file:
────────────────────────────────────────────────────────────────────────
curl -X POST http://localhost:5000/encrypt \
  -F "file=@document.pdf" \
  -F "policy=Role=Doctor AND Department=CSE"


📥 Decrypt with valid attributes:
────────────────────────────────────────────────────────────────────────
curl -X POST http://localhost:5000/decrypt \
  -H "Content-Type: application/json" \
  -d '{
    "encrypted_filename": "encrypted_document.pdf",
    "user_attributes": {"Role": "Doctor", "Department": "CSE"}
  }' \
  --output decrypted.pdf


📋 List all files:
────────────────────────────────────────────────────────────────────────
curl http://localhost:5000/list


📊 Get system metrics:
────────────────────────────────────────────────────────────────────────
curl http://localhost:5000/metrics


🎯 KEY FEATURES IMPLEMENTED
═══════════════════════════════════════════════════════════════════════

✅ Policy-Based Encryption
   Encrypt files with access policies like:
   • "Role=Doctor"
   • "Role=Doctor AND Department=CSE"
   • "Role=Admin OR Role=Manager"

✅ Attribute-Based Decryption
   Users provide attributes, system checks policy match:
   • Access granted if attributes satisfy policy
   • Access denied otherwise

✅ Dual Storage Support
   Toggle between:
   • Local: ./encrypted_files/
   • S3: AWS S3 bucket

✅ Resource Monitoring
   • CPU usage tracking
   • Memory usage tracking
   • Disk usage tracking
   • Operation time logging

✅ Production Ready
   • RESTful API
   • CORS enabled
   • Error handling
   • Comprehensive logging
   • CloudWatch compatible
