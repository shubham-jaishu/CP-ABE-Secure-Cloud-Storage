🎬 5-MINUTE DEMO SCRIPT
═══════════════════════════════════════════════════════════════

STEP 1: Create a file
─────────────────────
echo "Confidential patient data" > patient.txt

STEP 2: Encrypt with policy
────────────────────────────
curl -X POST http://localhost:8080/encrypt \
  -F "file=@patient.txt" \
  -F "policy=Role=Doctor AND Department=CSE"

SAY: "I'm setting a policy: ONLY doctors in CSE can access"

STEP 3: Try WRONG credentials
──────────────────────────────
curl -X POST http://localhost:8080/decrypt \
  -H "Content-Type: application/json" \
  -d '{"encrypted_filename": "encrypted_patient.txt", "user_attributes": {"Role": "Nurse"}}'

SAY: "Trying as a Nurse..."
RESULT: ❌ Access Denied!

STEP 4: Try CORRECT credentials
────────────────────────────────
curl -X POST http://localhost:8080/decrypt \
  -H "Content-Type: application/json" \
  -d '{"encrypted_filename": "encrypted_patient.txt", "user_attributes": {"Role": "Doctor", "Department": "CSE"}}' \
  --output decrypted.txt && cat decrypted.txt

SAY: "Trying as a Doctor in CSE..."
RESULT: ✅ Access Granted! File decrypted!

STEP 5: Show the web interface
───────────────────────────────
Open web_interface.html in browser
- Drag and drop files
- Set policies visually
- Test decryption with UI
